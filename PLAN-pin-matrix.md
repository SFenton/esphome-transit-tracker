## Phase 2 Collapse Animation

When the pin is triggered, the currently-displayed page has **V** trips
belonging to the newly-pinned route. Those V rows collapse (shrink to 0 height)
and replacement rows scroll up from below.

| Condition | V (collapsing rows) | Fill source |
|-----------|:-------------------:|-------------|
| T в‰¤ 3 (single page) | V = P (always) | None (display shrinks) |
| T > 3, pinned trips on current page | V = count on page (1 or 2) | Next trips in list |
| T > 3, pinned trips NOT on current page | V = 0 | No collapse animation |
| Last page, fill exhausted | V = P | Wraps to start of trip list |

## Collapse Scenarios by Page Position

For multi-page displays (T > 3), the number of collapsing rows depends on
which page you're viewing when the pin occurs. Trip ordering is by arrival time,
so the actual V depends on where the pinned route's trips fall in the sorted list.

### P = 1 (1 trip to pin)

| Current Page | V | Collapse Behavior |
|:------------:|:-:|-------------------|
| Page containing the trip | 1 | 1 row collapses, 1 fills from below (or wraps) |
| Any other page | 0 | No collapse; instant snap to split layout |

### P = 2 (2 trips to pin)

| Current Page | V | Collapse Behavior |
|:------------:|:-:|-------------------|
| Page with both trips | 2 | 2 rows collapse, 2 fill from below (or wrap) |
| Page with 1 trip | 1 | 1 row collapses, 1 fills from below (or wrap) |
| Page with 0 trips | 0 | No collapse; instant snap to split layout |

> Whether both trips of a 2-trip route land on the same page depends on their
> arrival times relative to other routes' trips. With rotate paging (window of 3),
> two trips at positions i and j are on the same page if |i в€’ j| < 3.
